# 候选池后二次压力突破监控

## 背景

上游系统发出的 `candidate_score_passed` 只表示项目评分合格。V2 不再把该事件直接当作买入信号，而是在 V2 信号模块中继续跟踪该 CA 的最新 K 线。当前默认数据源为 GMGN，保留 Birdeye 切换能力，等它真正出现回测中的“突破压力带买入”场景后，再向交易模块发出买入信号。

## 业务规则

- V2 信号模块订阅上游候选池 Redis Pub/Sub channel：`solana_scalper:candidate_pool`。
- 收到 `candidate_score_passed` 后，把 CA 写入 Redis 监控池，状态为 `watching`。
- 监控池每 2 秒轮询一次 active CA，调用 `datasource.kline_source` 对应的数据源查询 1m 最新 K 线；当前默认 GMGN。切换到 Birdeye 时该路径使用 Birdeye upstream，不使用项目级 K 线缓存。
- `watching` 状态下，如果配置了正数 `signal.min_market_cap`，且最新 K 线收盘市值小于该阈值，该 CA 从监控池移除；GMGN 价格源默认 `0`，不按市值阈值移除。
- `watching` 状态下，如果候选进入之后出现“突破压力带买入”场景，V2 发出标准 `TradeSignalMessage` 买入信号，`strategyCode=breakout_band_follow`，并把状态改成 `bought`。
- `bought` 状态继续监控卖点，即使最新值低于市值阈值也不移除，避免已买入仓位无人退出。
- 卖点复用回测 `breakout_band_follow` 规则：下一根跌回压力带上沿、硬止损、止盈、动态止损；实时监控不使用样本结束 timeout 卖出。
- 买入和卖出信号都发布到 V2 标准交易信号 Redis channel，由交易模块按当前 `paper/live` 模式执行。

## Redis 状态

- Active set：`solana_meme_v2:signal_monitor:active`
- 单 CA hash：`solana_meme_v2:signal_monitor:candidate:{ca}`
- 幂等 key：`solana_meme_v2:signal_monitor:emitted:{signalId}`

单 CA hash 记录 CA、symbol、上游 payload、候选进入时间、状态、最近一次成功拉到的当前市值、最近一次成功调用当前 K 线数据源拿到 K 线的时间、买入 signalId、买点、压力带信息等。服务重启后会从 active set 恢复未结束监控。

## 配置示例

```yaml
redis:
  enabled: true
  channel: "solana:meme:signals:pressure_breakout"
  consumer_channel: ""

datasource:
  kline_source: "gmgn"

gmgn:
  api_key: "${BACKTEST_GMGN_API_KEY}"
  max_qps: 8

signal:
  candidate_monitor_enabled: true
  candidate_channel: "solana_scalper:candidate_pool"
  poll_interval_seconds: 2
  interval: "1m"
  min_market_cap: 0
  lookback_bars: 120
  redis_key_prefix: "solana_meme_v2:signal_monitor"

trade:
  enabled: true
  signal_consumer: true
```

## 与交易模块的关系

交易模块只消费标准 `TradeSignalMessage`。候选池评分合格事件由信号模块处理，不再由交易模块直接转换成买入。

## 前端展示

- 回测工作台的交易面板新增 `Candidates` 页签，默认打开该页签。
- `Candidates` 页签展示当前仍在 Redis active 监控池中的上游候选项目，不展示已停止、已卖出或已被低市值过滤移除的历史项目。
- 表格字段包含状态、symbol、CA、上游评分、上游市值、当前市值、K 线获取相对时间、入池时间、买入信号和已触发买入后的压力带，不再展示策略/scan 列。
- 数据来自 `GET /api/signal/candidate-monitor`，该接口只读取 V2 信号监控池 Redis 状态，不额外查询其他项目源。
