# Solana Meme 回测系统 V2

## 背景

当前目录用于搭建新的 Solana 链上 meme 币回测系统。系统需要与现有 solana-scalper 项目技术栈保持一致，并支持后续嵌入其他系统。

## 第一版范围

- 展示 token 的 K 线数据。
- 支持用户录入或批量导入买入、卖出时间点。
- 按买卖点匹配 K 线收盘价，生成逐笔交易结果。
- 计算累计收益、胜率、最大回撤、交易次数、平均持仓时间。
- 将分析会话、买卖点、逐笔结果和指标快照保存到业务库。

## 设计约束

- 数据源通过接口抽象，第一版不绑定固定数据表。
- 新增 Birdeye K 线入口，显式选择 Birdeye 时只查询 Birdeye。
- Birdeye K 线在首次调用成功后必须写入本地 sqlite cache；后续同 token + interval 统一直接复用这个项目缓存，不再为追最新 K 线重复请求 Birdeye。
- 新增 CA + 钱包地址的买卖点标记入口，把该钱包在该 CA 上的 buy/sell 交易点标到 K 线上。
- 新增 Bitquery 交易点来源：K 线来自 Birdeye，钱包交易点来自 Bitquery。
- 新增数据库来源：K 线来自 `bar_data`，交易点来自 `trade_data`，按 `pair_id + user` 查询。
- 新增支撑/压力位计算：按局部高低点聚合价格带，并在 TradingView Lightweight Charts K 线上展示水平线。
- 支撑/压力位计算必须保持通用：只依赖传入 K 线 OHLCV，不绑定 `1m`、`5m`、`15m` 或 `1h` 等固定粒度；不同时间 K 线通过同一套算法计算。
- 价位强度需要综合触碰次数、价格带成交量、最近触碰时间和距离当前价远近；相近价位按容忍度合并成价格带。
- 支撑/压力角色按最新收盘价动态判定，已突破的历史压力位可以转为支撑，已跌破的历史支撑位可以转为压力。
- 压力突破场景定义为：在某个检测段内，`n` 根阳线的最高价落入同一个压力价格区间，且最后一次触及后的后续 `n` 根 K 线内收盘突破该压力区间；`n` 由前端的 `n值(阳线/后续K线)` 参数控制。
- 试压阳线除满足阳线且最高价进入压力带外，还要满足更高的成交量门槛；当前规则按最近 `volumeWindow` 根均量的至少 `1.35x` 与 `volumeMultiplier` 中较高者执行，较弱触碰不纳入试压点。
- 同一组试压阳线与最终突破阳线之间，最多只允许出现 `1` 根 K 线的最高价刺穿压力带上沿；如果中间超过 `1` 根 K 线已经上破压力带上沿，则视为结构已失真，不再算作这个试压突破场景。
- 压力突破计算拆成两类窗口：`窗口K线数` 用于生成 K 线检测窗口，`压力带窗口K线数` 和 `压力带窗口步长` 用于生成压力带窗口；后端对两组窗口做笛卡尔组合，组合后的时间交集没有 K 线则跳过，有 K 线则用压力带窗口产出的压力位去验证交集 K 线里的触压和后续突破。
- 压力突破场景按 K 线事件去重：同一组触及 K 线时间加同一个突破 K 线时间只保留一条结果；如果多个滑动窗口或相近压力带产出相同事件，保留强度分最高的一条。
- 压力突破场景的 K 线展示不绘制人工平台框；选中压力位后在图上显示压力带中心线、上下沿、参与聚类的 Pivot 点、触碰/试压点、`n` 根内突破点和买入点，保证压力位如何计算可以从 K 线上直接追踪。
- 前端 K 线图支持拖拽选择一个时间区域；选区产生精确的 K 线起止时间和 K 线数量，点击“计算选中区域压力位”后只用该时间段的 Birdeye K 线重新计算压力位、触碰点、突破点和买入点。
- 新增“按 CA 执行回测”能力：前端可直接对当前 CA + 时间区间执行策略回测，后端复用同一批 Birdeye K 线和滑动窗口结果，不重复请求上游。
- 回测方法必须通过独立注册表管理；页面先读方法列表，再按 `methodCode + methodConfig` 执行，便于后续继续新增其他回测方法。
- 当前内置方法为“突破压力带买入”：在突破压力带的 K 线收盘买入；若下一根 K 线跌破压力带上沿则止损；同时支持页面可配置硬止损，默认 `-5%`；若盈利先达到 5%，则后续回撤到 3% 收益时卖出；止盈比例和单笔投入金额由页面配置。
- 回测执行按单 token 单持仓处理：同一时间最多只允许持有一笔仓位；若前一笔尚未卖出，则后续新的买点信号全部跳过，只有在前一笔卖出后才允许再次买入。
- 当前内置方法新增止盈区间扫参：页面可输入例如 `8%-15%` 和步长，后端按每个止盈比例分别回测，并分组返回每组的盈亏、胜率、回撤与逐笔结果。
- 回测默认要计入手续费；当前默认每笔买入加卖出的总手续费比例为 `1.5%`，页面允许修改，收益统计全部按扣费后的净收益展示，同时保留毛收益和手续费拆分。
- 回测结果必须返回可回放的窗口编号、压力带、市值买卖点、卖出原因和收益，前端点击某笔交易要能跳回 K 线定位对应压力带；回测买卖点在图上用 `B/S` 标记，鼠标悬停可查看详情并显示对应成交价虚线。
- 全系统时间统一使用北京时间；接口入参和返回值使用 UTC+08:00，前端图表坐标、提示和表格时间均按北京时间展示。
- Birdeye 支持 API key 池；当单个 key 遇到 `429` 或额度耗尽时，后端自动切到下一个 key，避免因单 key 限流导致首次缓存一直失败。
- 不添加备用数据源、降级存储或隐式 mock 数据。
- 当前配置的数据源缺失、查询 SQL 未配置、Birdeye API Key 未配置或 Bitquery API Token 未配置时直接返回中文错误。
- 第一版不做滑点、手续费、成交失败和流动性仿真。

## 嵌入能力

- 前端 K 线图、买卖点编辑器、指标卡片和结果表格均拆成组件。
- 后端回测核心逻辑不依赖 HTTP 和数据库，可供其他入口复用。
- 未来嵌入其他系统时，优先新增 datasource 实现、TradePointDataSource 实现和宿主鉴权中间件，不改回测核心计算。
