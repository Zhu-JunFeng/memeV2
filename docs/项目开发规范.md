# 项目开发规范

本项目是 Solana meme 币回测系统 V2，技术栈与现有 solana-scalper 项目保持一致。

## 通用原则

- 后端使用 Go、Gin、GORM、Viper、Zerolog。
- 前端使用 Vue 3、Vite、Pinia、Vue Router、Element Plus、Axios、Vitest。
- 数据源必须通过接口抽象，不允许在回测核心逻辑中写死具体表结构或外部服务。
- 用户没有明确要求时，不要添加 fallback、备用数据源或降级存储。
- 功能变更必须补中文文档；接口变更同步更新 `docs/API.md`；DDL 变更同步更新 `docs/ddl/DDL变更记录.md`。

## 后端规范

- 接口返回统一结构：`code`、`message`、`data`、`traceId`。
- 回测计算逻辑放在 `backend/internal/backtest`，不得依赖 HTTP 或数据库实现。
- 数据读取放在 `backend/internal/datasource`，新增数据源时实现接口即可。
- 持久化放在 `backend/internal/repository`，不得参与收益计算。
- 本地测试优先执行：`cd backend && mkdir -p .tmp/go-build-cache && GOCACHE=$PWD/.tmp/go-build-cache go test ./...`。

## 前端规范

- 页面层负责编排，复用组件负责展示和局部交互。
- 可嵌入组件必须通过 props 接收数据，不直接绑定全局页面状态。
- API 调用统一通过 `frontend/src/api/http.js` 解包统一响应。
- 涉及组件逻辑时执行 `cd frontend && npm test && npm run build`。
